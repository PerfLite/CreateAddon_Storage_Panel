# Minecraft — Шпаргалка по ориентации блоков

## Система координат

```
        Y (вверх)
        |
        |
        +---------- X (восток, East)
       /
      /
     Z (юг, South)
```

| Ось | Направление + | Направление - |
|-----|--------------|--------------|
| X   | East (восток) | West (запад) |
| Y   | Up (вверх)    | Down (вниз)  |
| Z   | South (юг)    | North (север)|

---

## Стороны света (Direction)

| Direction | Ось | Знак | Куда смотрит |
|-----------|-----|------|--------------|
| NORTH     | Z   | -    | В экран (от игрока) |
| SOUTH     | Z   | +    | На игрока |
| EAST      | X   | +    | Вправо |
| WEST      | X   | -    | Влево |
| UP        | Y   | +    | Вверх |
| DOWN      | Y   | -    | Вниз |

> **Запомни:** Когда стоишь лицом на NORTH — Z уменьшается. SOUTH — Z растёт.

---

## Повороты модели в blockstate (y rotation)

Базовая модель всегда смотрит на **SOUTH** (y=0).

| y поворот | Лицо модели смотрит на |
|-----------|----------------------|
| 0         | SOUTH (юг, +Z)       |
| 90        | WEST (запад, -X)     |
| 180       | NORTH (север, -Z)    |
| 270       | EAST (восток, +X)    |

```
        NORTH (y=180)
           ↑
WEST  ←    +    → EAST
(y=90)     ↓    (y=270)
        SOUTH (y=0)
```

## Повороты по X (для пола/потолка)

| x поворот | Результат |
|-----------|-----------|
| 0         | Вертикально (стена) |
| 90        | Лицо смотрит вниз (потолок) |
| 270 (-90) | Лицо смотрит вверх (пол) |

## Модель блока — координаты граней

Блок занимает пространство от `[0,0,0]` до `[16,16,16]`.

```
  Грань NORTH = z=0  (ближняя к северу)
  Грань SOUTH = z=16 (дальняя, к югу)
  Грань WEST  = x=0
  Грань EAST  = x=16
  Грань DOWN  = y=0
  Грань UP    = y=16
```

Наша панель: `from=[0,0,14] to=[16,16,16]`
- Лицо (front) = грань NORTH на z=14 → смотрит в -Z (на север)
- При y=0 лицо смотрит на **юг** (потому что модель повёрнута)

## Как работает facing в нашем моде

`FACING` = направление **куда смотрит лицо** панели.

| facing | y в blockstate | Лицо смотрит | Панель прикреплена к |
|--------|---------------|--------------|---------------------|
| NORTH  | 0             | Юг (SOUTH)   | Северной стене      |
| SOUTH  | 180           | Север (NORTH)| Южной стене         |
| EAST   | 90            | Запад (WEST) | Восточной стене     |
| WEST   | 270           | Восток (EAST)| Западной стене      |

> **Vault/Tank** находится **за стеной** = в направлении `facing.getOpposite()`

## Позиция лицевой грани в рендерере

```java
final float F = 14f / 16f + 0.002f;  // = 0.877 (дальняя грань)
final float B = 1f - F;               // = 0.123 (ближняя грань)
```

| facing | tx    | tz    | yRot  |
|--------|-------|-------|-------|
| NORTH  | 0.5   | B     | 180°  |
| SOUTH  | 0.5   | F     | 0°    |
| WEST   | B     | 0.5   | 270°  |
| EAST   | F     | 0.5   | 90°   |

## Быстрая проверка blockstate

Если модель **не там где хитбокс**:
1. Проверь `y` поворот в blockstate — он должен совпадать с `yRot` в рендерере
2. Проверь `tx/tz` в рендерере — они должны указывать на лицевую грань модели

Если текст **зеркалится**:
- Поменяй знак у одной из осей в `poseStack.scale()`
- Или убери/добавь `mulPose(Axis.YP.rotationDegrees(180f))`

## F3 подсказки в игре

- **F3** — показывает координаты и направление взгляда
- **Facing** в F3 — куда смотрит игрок (NORTH/SOUTH/EAST/WEST)
- **XYZ** — текущие координаты
- **/function compass:enable** — наш HUD со сторонами света
